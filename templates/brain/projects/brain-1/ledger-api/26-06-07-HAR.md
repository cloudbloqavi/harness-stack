# ledger-api — 26-06-07  (detailed log)

## 7f3a9c1 — Add /v2/transactions endpoint with cursor pagination

**What changed:** Added `GET /v2/transactions` returning cursor-paginated
entries (default page size 100, `next_cursor` in the envelope). Deprecated the
unpaginated `/v1/transactions`, which now returns a `Deprecation` header and is
scheduled for removal in 90 days.

**Why:** `/v1` serialised the entire ledger in one payload and was timing out
for large accounts. Cursor pagination caps response size and removes the
timeout without a breaking change to the data model.

**Files:** `src/routes/transactions.ts`, `src/pagination.ts`, `openapi.yaml`

**Cross-repo impact:** `ledger-web` (same brain) must migrate off `/v1` before
the 90-day sunset — tracked in its detailed log the same day (b21e0d4).

**Flags / to-dos:** Remove the `/v1` route after the sunset window; add metrics
on `/v2` cursor depth to tune the default page size.

## a9f0c12 — Add Deprecation and Sunset headers to /v1/transactions

**What changed:** `/v1/transactions` now returns RFC 8594 `Deprecation: true`
and `Sunset: Sat, 05 Sep 2026 00:00:00 GMT` headers, plus a `Link` header
pointing clients at the `/v2` docs.

**Why:** Follow-up to `7f3a9c1` — machine-readable deprecation signals let
`ledger-web` (and any external caller) detect the sunset programmatically
instead of relying on the changelog.

**Files:** `src/routes/transactions.ts`, `src/middleware/deprecation.ts`

**Cross-repo impact:** `ledger-web` can key its `/v1` fallback removal off the
`Sunset` date rather than hard-coding it.

**Flags / to-dos:** None.
